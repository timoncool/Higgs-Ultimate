API Info

Examples use the current Base URL and API key.

Endpoints

GET /healthNo auth. Checks whether the local server is alive.
GET /v1/statusEngine, model, and generation state.
GET /v1/modelsDetected local Higgs model folders.
GET /v1/higgs/speakersSaved speaker identities available as speaker:<id> voices.
POST /v1/audio/speechPlain TTS with the loaded model.
POST /v1/higgs/audio/streamStreaming TTS/clone/continue as newline JSON events with wav-base64 chunks.
POST /v1/higgs/voice-cloneTTS with a reference audio file path.
POST /v1/higgs/continue-speechContinue an existing audio file.
POST /v1/higgs/cancelCancel the active generation.

Request Options

AuthorizationBearer API_KEY header for every /v1 route.
inputText to speak. Required for speech and voice clone.
voicedefault for plain TTS, or speaker:<id> to use a saved speaker identity.
reference_audio_pathLocal path to WAV, MP3, FLAC, or M4A for voice clone.
reference_textOptional transcript for the reference voice.
audio_pathLocal path to the audio file used by continue speech.
continuation_textOptional text to speak after the input audio.
response_formatFinished-file API routes accept wav or mp3. Streaming sends live wavBase64 chunks and returns final wavBase64 or mp3Base64.
stream events/higgs/audio/stream returns queued, start, progress, audio, final, done, or error NDJSON events. Decode audio.wavBase64 chunks for live playback, then save final.wavBase64 or final.mp3Base64.
max_tokensGeneration length limit.
temperatureRandomness. Lower is steadier, higher is more varied.
top_pNucleus sampling filter.
top_kTop token count filter.
seedFixed seed for repeatability.
